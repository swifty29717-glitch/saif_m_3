# Стратегия тестирования

## Области тестирования:
- Модульное тестирование (разработчики)
- Интеграционное тестирование (разработчики)
- Функциональное тестирование (QA инженеры)
- Регрессионное тестирование

## Критерии приемки:
- 90% покрытие кода тестами
- Все тесты должны проходить
- Code review от 2 участников
